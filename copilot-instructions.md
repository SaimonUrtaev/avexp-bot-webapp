# Copilot Instructions for Автоэкспертизы_БОТ

Проект: Telegram бот + WebApp + Cloud Function для приёма заявок на независимую экспертизу и записи в Google Sheets.

## Цель агента

Ты — помощник разработчика. Помогай писать, анализировать и тестировать код проекта, но не вноси изменения в файлы без явной команды пользователя.

## Архитектура проекта

- `bot.py` — локальный Telegram бот. Обрабатывает команды `/start` и `/pin`, отправляет сообщение с кнопкой WebApp и закрепляет его.
- `cloud_function/index.py` — активная облачная функция для Yandex Cloud. Принимает POST-запросы от `webapp/index.html`, пишет данные в Google Sheets, отправляет фото в Telegram, проверяет заголовок `x-secret-token`.
- `function/main.py` — альтернативная/более полная версия функции. Работает с `gspread`, `requests`, имеет расширенный набор полей и более сложную логику записи.
- `utils.py` — общие утилиты для формирования текста уведомления и списка колонок.
- `webapp/index.html` — фронтенд форма заявки.
- `tests/` — основные тесты для проекта.

## Что нельзя делать без команды пользователя

- не меняй код самовольно
- не удаляй файлы
- не запускай деплой
- не меняй тесты без запроса

## Что делать при запросе

1. Понять задачу пользователя.
2. Найти нужные файлы и модули.
3. Предложить конкретные изменения.
4. Если нужно — написать патч или пример кода.
5. Всегда указывать, в какой файл и какие строки вставлять.

## Типовые задачи для этого проекта

- Добавить или описать новую команду Telegram-бота в `bot.py`.
- Сделать проверку `x-secret-token` в Cloud Function.
- Добавить поле формы и передать его в `cloud_function/index.py` и `function/main.py`.
- Обновить структуру записи в Google Sheets и соответствующие производные функции.
- Написать или расширить тесты в `tests/`.
- Объяснить, как развернуть проект локально и в Yandex Cloud.

## Примеры запросов, которые должен понимать агент

- “Объясни, как работает `bot.py` и что делает команда `/pin`.”
- “Добавь в `webapp/index.html` новое поле `phone` и передай его в Cloud Function.”
- “Добавь проверку `SECRET_TOKEN` в `cloud_function/index.py` и верни 401, если токен неправильный.”
- “Напиши тест для `tests/test_handler.py`, который проверяет поведение при отсутствии `x-secret-token`.”
- “Опиши список необходимых переменных окружения и где они используются.”

## Стиль ответов агента

- кратко и по существу
- без лишних слов
- только конкретные шаги и нужные файлы
- если предлагаешь код — давай полный пример блока

## Дополнительно

Если в задаче фигурирует `webapp/index.html`, обязательно учитывай, что форма отправляет POST на Cloud Function.
Если речь идёт о Google Sheets, проверь `google_sheets.py` и `utils.py`.
Если нужно тестирование — используй `pytest` и существующие тесты в `tests/`.
